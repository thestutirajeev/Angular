HTML View

The HTML container where the AngularJS application is displayed, is called the view.

The view has access to the model, and there are several ways of displaying model data in the view.

    One-way binding means that changes in the model update the view, but changes in the view do not update the model.
  1. You can use the ng-bind directive, which will bind the innerHTML of the element to the specified model property:

    2. Name:Change the name inside the input field, and the model data will not change automatically, and therefore the above value will not change because of one-way binding.

    The ng-bind directive provides a one-way binding between the model and the view.

  2. You can use the double curly braces {{ }} to bind the model to the view. This is called interpolation:

    3. {{firstname}}

    The double curly braces {{ }} provide a one-way binding between the model and the view.

  3. You can use the ng-model directive to bind the model to the view. The ng-model directive binds the value of HTML controls (input, select, textarea) to application data:

    4. The ng-model directive provides a two-way binding between the model and the view.

AngularJS Controller

  • Applications in AngularJS are controlled by controllers.
  • The ng-controller directive defines the application controller.
  • Because of the immediate synchronization of the model and the view, the controller can be completely separated from the view, and simply concentrate on the model data. Thanks to the data binding in AngularJS, the view will reflect any changes made in the controller.
